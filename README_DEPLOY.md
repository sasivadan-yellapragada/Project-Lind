# Project Lind Render Deployment

## Project Structure

```text
.
├── backend/
│   ├── index.js          # Express server and API routes
│   ├── db.js             # SQLite connections and schema initialization
│   ├── ai.js             # Hybrid Q&A retrieval
│   └── package.json      # Backend start script and dependencies
├── frontend/
│   ├── src/              # React application
│   ├── dist/             # Production build output, generated by Vite
│   └── package.json      # Frontend build script
├── clinical_trials.py    # Optional local ingestion CLI
├── ai_build_corpus.py    # Optional AI corpus builder
├── package.json          # Root Render build/start scripts
├── render.yaml           # Render Blueprint configuration
├── trials.db             # Local source SQLite DB, ignored by Git
└── backend/ai_corpus.db  # Local AI retrieval DB, ignored by Git
```

## Render Deployment Model

This project is configured as one Render web service:

```text
User
  -> Render URL
  -> Express server
  -> React static frontend
  -> /api routes
  -> SQLite databases
```

Express serves both the React app and the API. The frontend calls `/api` on the same origin, so no production CORS configuration is required. Hybrid Q&A uses `AI_DB_PATH` to read a generated retrieval corpus; it does not call a hosted LLM.

## Render Settings

Use these values if creating the service manually:

```text
Root Directory: .
Build Command: npm run build
Start Command: npm start
```

The included `render.yaml` uses the same settings and can be used as a Render Blueprint.

## Environment Variables

```text
NODE_ENV=production
DATA_DIR=/var/data
SOURCE_DB_PATH=/var/data/trials.db
USER_DB_PATH=/var/data/user_data.db
AI_DB_PATH=/var/data/ai_corpus.db
```

Optional:

```text
VITE_API_BASE_URL=/api
```

`VITE_API_BASE_URL` is optional because the frontend defaults to `/api`.

## Database Notes

The app currently uses SQLite.

- `trials.db` is the read-only source clinical trials database.
- `ai_corpus.db` is the generated Hybrid Q&A retrieval database.
- `user_data.db` stores watchlist, notes, and tags.
- `trials.db` and `ai_corpus.db` are ignored by Git because they are generated local database artifacts.
- If `trials.db` is missing on Render, the backend creates an empty source schema so the app still boots.
- If `ai_corpus.db` is missing on Render, `/api/ask` returns a grounded refusal until the corpus is generated or uploaded.
- User data and generated databases should be stored on a Render persistent disk through `/var/data`.

For a production data set on Render, either:

1. Attach a persistent disk and upload or generate `/var/data/trials.db` and `/var/data/ai_corpus.db`.
2. Migrate source trials, AI chunks, and user data to a managed database such as Postgres.

Without a populated `trials.db`, the deployed UI works but the trial list will be empty. Without `ai_corpus.db`, Hybrid Q&A will decline to answer.

## Build Generated Databases

Create source trial data locally:

```bash
python clinical_trials.py ingest --condition "non-small cell lung cancer"
```

Create the AI corpus locally:

```bash
npm run build:ai
```

For a trial-only AI corpus without openFDA calls:

```bash
python3 ai_build_corpus.py --skip-openfda
```

Upload the generated SQLite files to the Render persistent disk paths configured by `SOURCE_DB_PATH` and `AI_DB_PATH`.

## Common Deployment Issues

### Empty Trial List

Cause: `trials.db` is not present in the deployed environment.

Fix: populate `trials.db` on a persistent disk or migrate the data into a hosted database.

### Hybrid Q&A Refuses Every Question

Cause: `AI_DB_PATH` does not point to a populated `ai_corpus.db`.

Fix: build and upload `ai_corpus.db`, or run `npm run build:ai` against the deployed persistent disk if your deployment workflow supports that.

### Watchlist/Notes/Tags Reset

Cause: `USER_DB_PATH` points to ephemeral storage.

Fix: use a Render persistent disk and set:

```text
USER_DB_PATH=/var/data/user_data.db
```

### API Calls Go To localhost

Cause: frontend was built with a hardcoded local API URL.

Fix: this project now defaults to `/api`. Rebuild the frontend with:

```text
npm --prefix frontend run build
```

### Deep Links 404

Cause: Express is not serving the React SPA fallback.

Fix: this project now sends all non-API routes to `frontend/dist/index.html`.

## Redeploy

After committing and pushing changes:

```bash
git push origin main
```

Render will rebuild automatically if auto-deploy is enabled.

Manual redeploy:

1. Open the Render service.
2. Click **Manual Deploy**.
3. Choose **Deploy latest commit**.
